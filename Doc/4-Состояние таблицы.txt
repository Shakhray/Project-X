Состояние таблицы. Общий вид. Возможная система обозначений.

1. Операции.
 
   1. Добавление.  |
   2. Изменение.   | Столбца, записи, первичного ключа.
   3. Удаление.    |

   4. Отмена.      | Операции, всех изменений.
   5. Сохранение.

2. Состояние.

   1. Структура (таблицы).
      Набор столбцов. Рассматриваем как упорядоченный.
         Столбец: идентификатор (имя), тип данных БД, значения по умолчанию.
      Первичный ключ.

   2. Данные.
      Набор записей.
         Запись: набор значений.

      значения в записи соответствуют столбцам. Соответствие задаётся порядком.

   Свойства и отношения.
   1. Представимость значения типом.
   2. Определённость: значения, столбца, данных, структуры, ...
   3. Обязательная определённость: элемента, значения.
   4. Обязательность элемента. Или необязательность.
   5. Пустота: структуры, данных.
   6. Наличие элемента.

======================================================================================
Возможные обозначения.

Важные требования:
   1. Выражения должны записываться в строку.
   2. Должны использоваться наиболее распространённые (простые) символы, чтобы можно
      было использовать систему обозначений в простых текстовых файлах.

Базовые обозначения.
   1. Элемент декартова рпоизведения - стандартное обозначение.
   2. Индексы: отделяем подчёркиванием. Например, b_i, b_ij.
   3. Последовательность: {a_k} (в фигурных скобках).
   4. k изменяется от k_1 до k_2: k ~ [k_1, .., k_2].
   5. Если отношение записывается несколькими символами (особенно, буквами), то записываем
      его в квадратных скобках.
      То же относится к свойствам.
      Например: v [exp] t - отношение между v и t.
                [Def] v - свойство v.

   6. Если элемент обозначается несколькими символами, записываем его между вертикальными чёрточками.
      Например: |PK|.

Обозначения для состояния таблицы.

Tab - таблица.
|Tab| = (S, D)
S = ({c_i} i ~ [1, .., n], |PK|)            c_i - столбцы, n - количество столбцов, |PK| - первичный ключ
     c_i = (N_i, t_i)                       N - имя столбца, t - тип столбца.
     |PK| = {c_i_k} k ~ [1, .., m], m <= n  

D = {{v_ji}} j ~ [1, .., r], i ~ [1, .., n]    r - количество записей.
    r_j = {v_ji} i ~ [1, .., n] - запись с номером j.

Отношения и свойства.
1. Значение v представимо типом t: v [exp] t
2. Определённость значения:        [Def] v
3. Обязательная определённость:    [MDef]
4. наличие элемента:               [Ex]
5. Обязательность элемента:        [MEx]

Замечание: некоторые нужно обозначать внутри структуры.


Недостатки этой системы обозначений:
1. Недостаточная системность, особенно базовых обозначений.
2. Неудобные обозначения для индексов с нечёткой семантикой.
3. Обозначения свойств не итегрированы в представление состояния таблицы.
4. Использование круглых скобок.
